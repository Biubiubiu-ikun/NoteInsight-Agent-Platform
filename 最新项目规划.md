# NoteInsight Agent Platform 最新项目规划

> 版本：V7.1，更新日期：2026-07-18
> 本文件是项目唯一权威规划。所有以“旧版”开头或带旧版本号的规划仅作历史参考；冲突内容以本文为准。

## 1. 产品目标

NoteInsight 面向内容平台创作者与运营人员，从图文笔记、OCR、评论和行为事实中提取可追溯证据，完成检索、分析和有引用的洞察报告。

核心闭环：

`账号与内容社区 -> 行为事件 -> 日事实 -> Evidence Store -> 混合检索 -> 离线评测 -> 洞察 Agent -> 可视化工作台`

## 2. 技术基线

- Backend：Go、Gin、sqlx、PostgreSQL 16，不引入 GORM。
- Cache/Ranking：Redis 7。
- Async：Transactional Outbox、NATS JetStream、幂等消费、DLQ。
- Frontend：React、TypeScript、Vite、Lucide。
- Quality：Go test/race/vet、真实 PostgreSQL/NATS 集成测试、Vitest 覆盖率门槛、Playwright E2E、Compose acceptance、OpenAPI 路由漂移、CodeQL SARIF 门禁、依赖与镜像审计。
- Observability：Prometheus、Grafana、结构化日志、request/trace correlation、告警规则。
- 代码边界：handler 只解析参数、读取鉴权上下文和响应；规则在 service，SQL 在 repository。

## 4. Phase 6C 完成基线

### 4.1 数据与权限边界

- 已建立 `projects`、`project_members`、`datasets`、`dataset_notes`。
- `notes` 支持 `public/project` 可见性、`content_version` 和软删除。
- 每个项目自动获得 `community` 数据集，新用户自动加入默认公共项目。
- 内容创建校验项目写权限；读取按项目成员与可见性过滤。
- `evidence_sources` 已回填 note、media、comment，保存项目、数据集、版本、SHA-256、可见性、索引状态和删除状态。
- Evidence Source 触发器覆盖创建、内容版本切换、评论软删除和父笔记删除传播。

### 4.2 一致性与事件

- `note.updated`、`note.deleted` 与业务更新在同一事务写 Outbox。
- 登录用户读取详情产生 `note.viewed`；迟到的浏览事件遇到已删笔记时保留行为事实并幂等忽略派生计数。
- Event Envelope 固定 `schema_version=1`，包含 producer、correlation_id、trace_id。
- Worker 拒绝未知版本；互动增删与评论点赞继续由唯一约束和 processed event 保证幂等。
- Counter reconcile 使用循环批次检查点，排名 SQL 只返回需要的 Top-N。

### 4.3 账号与安全

- Access Token 默认 30 分钟；生产环境最长 2 小时。
- Refresh Token 一次一换且只存 hash；旧 Token 重用撤销活动会话。
- 浏览器 Refresh Token 使用 HttpOnly/SameSite Cookie，前端不写 localStorage。
- 认证入口按 IP 限流，内容写入按 user 限流。
- CORS 来源和可信代理必须显式配置；生产禁止通配来源。
- 管理员/作者修改生成结构化 audit mutation 日志。

### 4.4 运维与恢复

- migration 使用 PostgreSQL advisory lock 和 SHA-256 checksum，连续执行稳定跳过。
- 提供 maintenance、materialize、reconcile、dlqctl、backup/restore 命令。
- Prometheus/Grafana 配置覆盖 HTTP、DB pool、Outbox、JetStream、event lag 与 reconcile。
- 2026-07-15 已完成 custom-format 备份与隔离恢复演练；恢复库完成表计数校验后销毁。
- 数据删除、保留、PII、检索过滤和审计规则见 `docs/14_data_governance.md`。

### 4.5 内容与前端闭环

- 支持 like/unlike、collect/uncollect、comment like/unlike。
- Note 返回 author projection 与 viewer interaction state。
- Feed 使用服务端查询、摘要与 keyset pagination；热榜一次返回 Note 摘要，消除前端 N+1。
- 详情使用 `/notes/{id}` 深链接，分享链接可直接打开目标笔记。

## 7. Phase 7B：词法检索与离线评测（已完成）

## 7.2 Phase 7D：检索可信度与生产加固（下一阶段）

1. 冻结 stratified benchmark v5：每个 task 覆盖 development/holdout，等价证据使用 multi-Gold/relevance pool，独立复核拒答并记录一致性；增加乱码、随机串和领域外查询，单独校准 dense OOD 拒答。

4. v5 冻结后再评估 pinned cross-encoder reranker，报告逐任务收益、延迟和成本；无显著收益则不引入。

## 8. Phase 8：Insight Agent

1. 单 Agent 流程：意图解析 -> 检索计划 -> 证据收集 -> 分析 -> 引用校验 -> 报告。
2. 工具调用具备超时、重试、预算、取消和 trace。
3. 输出受众画像、主题趋势、争议点、反馈、机会与风险。
4. 每个事实性结论必须关联 Evidence Source/Citation。
5. 保存 prompt、model、tool、run 和评测 lineage。
